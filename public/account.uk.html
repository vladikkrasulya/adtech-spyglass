Що відстежується, а що ні

Spyglass — zero-knowledge для payload'ів. Статистика «Активність» у кабінеті береться з маленького per-user логу що зберігає ЛИШЕ метадані нижче. Тіла bid request і bid response НІКОЛИ не зберігаються до цього логу — вони лише проходять через валідатор, findings виходять, тіла відкидаються.

Зберігається timestamp · oRTB версія · статус (clean/warnings/errors) · формат (banner/video/native/...) · кількість findings (errors / warnings / total)
Не зберігається тіла payload · ID окремих findings · IP · user-agent · ідентифікатор партнера · timestamps з точністю меншою за день у агрегатах
Анонімні дзвінки не логуються
Видалення акаунту лог видаляється каскадно (foreign key)

Корпус поведінки

Збережені event-streams з behavior-вкладки, помічені як легітимні / шахрайські / неоднозначні. Confusion matrix нижче рахує precision/recall кожного детектора на твоєму ж корпусі.

Confusion matrix · precision і recall

Кожен патерн прогоняється через corpus як fraud-детектор: TP — спрацював на «шахрайство», FP — спрацював на «легітимно», FN — пропустив «шахрайство». Сорт по F1.

Шифрування і відновлення

Бібліотека зашифрована ключем що деривується з твого пароля. Recovery key (32-символьний hex) показували один раз при реєстрації — тримай його у password manager. Якщо втратиш обидва — бібліотеку можна лише стерти, не відновити.
