## Known false positives (do not action)

- 🔴 ".env file present in repository root" — gitignored; standard practice; not debt.
- 🟡 "Duplicated about pages per locale" — by design per 3-locale parity hard-rule.
- 🟡 "No visible rate limiting on sensitive endpoints" — rate limits ARE in place (auth.js).
- 🟡 "i18n key drift en=192 uk/ru=191" — the missing key is `_note` (metadata sentinel), not a real translation.

---

### 🔴 CRITICAL: `.env` file present in repository root

**Evidence:** file tree includes `.env` (515 bytes) alongside `.env.example`
**Why it's debt:** If tracked by git, secrets (API keys, tokens) are exposed to anyone with repository access. Even if untracked, its presence in the working tree risks accidental commit.
**Fix direction:** Ensure `.env` is in `.gitignore`, remove any existing tracking (`git rm --cached .env`), and audit commit history for leaked secrets.

### 🟠 HIGH: Missing Content Security Policy (CSP) headers

**Evidence:** No CSP configuration found in snapshot (no `helmet`, no manual `Content-Security-Policy` header in `server.js` or modules)
**Why it's debt:** Without CSP, the app is vulnerable to XSS attacks, especially given dynamic content from bid responses and user-generated pastes.
**Fix direction:** Add `helmet` middleware or set a strict CSP header in the HTTP response pipeline.

### 🟡 MEDIUM: 108 ESLint warnings (mostly unused `catch` variables)

**Evidence:** Lint output shows 108 warnings across 18 files, e.g. `server.js:269:14  warning  'e' is defined but never used`, `public/spyglass.app.js:158:14`, `public/modules/intel/index.js:155:16`, etc.
**Why it's debt:** Warnings mask real issues, increase noise, and unused catch blocks may silently swallow errors that should be logged or handled.
**Fix direction:** Rename unused catch variables to `_e` or remove the catch block if truly unnecessary; add error logging where appropriate.

### 🟡 MEDIUM: Console logging left in production code

**Evidence:** `packages/core/decoders/index.js:61` and `:69` have `// eslint-disable-next-line no-console`; `packages/core/rules/index.js:55` has the same
**Why it's debt:** `console.log` in production can leak internal state, clutter stdout, and lacks log levels or structured output.
**Fix direction:** Replace with a proper logging library (e.g., `pino`) or guard with `process.env.NODE_ENV !== 'production'`.

### 🟡 MEDIUM: Manual cache-busting via `ASSET_VERSION` in inspector module

**Evidence:** `public/modules/inspector/index.js:37` defines `const ASSET_VERSION = '22';` and uses it on lines 62, 89 for template and CSS URLs
**Why it's debt:** Manual version bumps are error-prone; forgetting to increment after a change causes stale caches for users.
**Fix direction:** Implement content-hash-based asset versioning at build time or use a query string based on file mtime.

### 🟡 MEDIUM: Monolithic frontend file `spyglass.app.js` (4505 lines)

**Evidence:** File size list shows `./public/spyglass.app.js` at 4505 lines, far larger than any other frontend module
**Why it's debt:** A single huge file is hard to navigate, test, and split; it concentrates too many concerns and increases merge conflict risk.
**Fix direction:** Continue extracting domain logic into `public/modules/` and leave only bootstrapping/coordination in the main app file.

### 🟡 MEDIUM: Missing i18n key in Ukrainian and Russian locales

**Evidence:** `packages/core/messages/en.json` has 195 keys, while `uk.json` and `ru.json` have 194 keys each
**Why it's debt:** One validator message will fall back to English for Ukrainian/Russian users, degrading the localized experience.
**Fix direction:** Identify the missing key (likely a recently added rule) and add translations for `uk` and `ru`.

### 🟡 MEDIUM: No test coverage enforcement in CI

**Evidence:** `package.json` scripts: `"ci": "npm run format:check && npm run lint && npm run typecheck && npm test"` — no coverage flag or threshold
**Why it's debt:** Without coverage requirements, new code may lack tests, and regressions can slip through unnoticed.
**Fix direction:** Add `--experimental-test-coverage` to the test command in CI and set a minimum coverage threshold (e.g., 80%).

### 🟡 MEDIUM: No structured logging

**Evidence:** Previous audit noted no structured logging; current code still uses `console.log` (see eslint-disable comments) and no logging library is present in `dependencies`
**Why it's debt:** Production debugging is difficult without log levels, timestamps, and structured output; critical errors may be missed.
**Fix direction:** Integrate a lightweight logger like `pino` and replace all `console.*` calls with leveled logging.

### 🟡 MEDIUM: No error tracking (e.g., Sentry)

**Evidence:** No error-tracking dependency or initialization code found; previous audit flagged this as remaining MEDIUM debt
**Why it's debt:** Unhandled exceptions and API failures in production go unnoticed, delaying fixes and impacting users.
**Fix direction:** Add an error-tracking service (Sentry, etc.) and capture uncaught exceptions and promise rejections.

### 🟡 MEDIUM: Duplicated about pages per locale

**Evidence:** `public/about.en.html` (1345 lines), `about.ru.html` (1347), `about.uk.html` (1350) — nearly identical large files
**Why it's debt:** Any content change must be replicated across three files, risking inconsistency and increasing maintenance burden.
**Fix direction:** Consolidate into a single template with i18n string injection, similar to the inspector module.

### 🟡 MEDIUM: Duplicated inspector templates per locale

**Evidence:** `public/modules/inspector/template.en.html` (581 lines), `template.ru.html` (601), `template.uk.html` (604)
**Why it's debt:** Same as above — structural HTML duplicated with only text differences; high risk of drift.
**Fix direction:** Use a single template and inject localized strings via the existing i18n system.

### 🟡 MEDIUM: No visible rate limiting on sensitive endpoints

**Evidence:** No rate-limit middleware (e.g., `express-rate-limit`) found in `server.js` or module handlers; auth endpoints (`/auth/*`) are exposed
**Why it's debt:** Brute-force attacks against login, password reset, or token endpoints are possible without throttling.
**Fix direction:** Add rate limiting to authentication and other sensitive routes, possibly via a small middleware or in the router.

### 🟢 LOW: Unused imports/variables in `server.js`

**Evidence:** Lint warnings: `'https' is assigned a value but never used` (line 34), `'readJson'`, `'makeError'`, `'MAX_BODY_BYTES'` (line 45), `'PROXY_ALLOWED_HOSTS'` (line 610)
**Why it's debt:** Dead code clutters the file, may confuse maintainers, and suggests incomplete cleanup after the modular migration.
**Fix direction:** Remove the unused imports and variable declarations.

### 🟢 LOW: Unused variable `escapeHtml` in auth frontend module

**Evidence:** `public/modules/auth/index.js:57:13  warning  'escapeHtml' is defined but never used`
**Why it's debt:** Unused imports increase bundle size (if not tree-shaken) and mislead readers about what the module depends on.
**Fix direction:** Remove the import if truly unused, or use it where intended.

### 🟢 LOW: Unused variable `readPendingRecovery` in recovery module

**Evidence:** `public/modules/recovery/index.js:78:10  warning  'readPendingRecovery' is defined but never used`
**Why it's debt:** Dead code suggests incomplete feature or refactor leftover; may confuse future developers.
**Fix direction:** Remove the unused function or complete the feature that uses it.

### 🟢 LOW: Unused variables in test files

**Evidence:** `tests/api-stability.test.js:19` (`LEVELS`), `tests/mirror.test.js:18` (`validate`, `crosscheck`), `tests/replay.test.js:14,147` (`p`)
**Why it's debt:** Unused variables in tests may indicate missing assertions or leftover setup; they reduce clarity.
**Fix direction:** Remove the unused variables or incorporate them into assertions.

### 🟢 LOW: Unused variables `rect` and `badge` in `spyglass.app.js`

**Evidence:** Lint warnings: `'rect' is assigned a value but never used` (line 412), `'badge' is assigned a value but never used` (line 347)
**Why it's debt:** Dead assignments suggest incomplete UI logic or leftover code from refactoring.
**Fix direction:** Remove the assignments or complete the intended functionality.

### 🟢 LOW: `eslint-disable` for `no-func-assign` in account.js

**Evidence:** `public/account.js:766:  // eslint-disable-next-line no-func-assign -- intentional decorator pattern`
**Why it's debt:** Reassigning a function is a code smell that can confuse static analysis and developers; the decorator pattern can be implemented more cleanly.
**Fix direction:** Refactor to use a wrapper function or a proper decorator pattern without reassignment.

### 🟢 LOW: Large test files could be split

**Evidence:** `tests/behavior.test.js` (920 lines), `tests/validator.test.js` (907 lines)
**Why it's debt:** Large test files are harder to navigate and maintain; failures may be less granular.
**Fix direction:** Split into smaller test suites organized by feature or rule group.

### 🟢 LOW: `ARCHMAP.md` is 31 KB and may become stale

**Evidence:** `docs/ARCHMAP.md` is 31410 bytes, one of the largest docs
**Why it's debt:** A large architectural map requires constant manual updates; if neglected, it misleads new contributors.
**Fix direction:** Consider auto-generating parts of it from code structure or splitting into smaller focused docs.
